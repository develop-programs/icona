# Icona Autocomplete & Search Guide\n\nThis guide explains how to use Icona's autocomplete functionality, similar to Lucide icons.\n\n## Features\n\n\u2705 **TypeScript Autocomplete**: Full TypeScript support with icon name suggestions  \n\u2705 **Fuzzy Search**: Find icons even with partial or misspelled queries  \n\u2705 **Keyword Matching**: Search by functionality (e.g., \"navigation\", \"communication\")  \n\u2705 **Category Filtering**: Browse icons by collection (Hicon, waveicons, etc.)  \n\u2705 **Tag-based Search**: Find icons by semantic tags  \n\u2705 **Smart Scoring**: Results ranked by relevance and popularity  \n\u2705 **JSDoc Documentation**: Rich tooltips in your IDE  \n\n## Quick Start\n\n### 1. Import with Autocomplete\n\n```typescript\n// TypeScript will suggest icon names as you type\nimport { ChevronRight, Home, Search } from 'icona/react';\n\n// Or for Vue\nimport { ChevronRight, Home, Search } from 'icona/vue';\n\n// Or for React Native\nimport { ChevronRight, Home, Search } from 'icona/react-native';\n```\n\n### 2. Search Icons Programmatically\n\n```typescript\nimport { getIconSuggestions, searchIcons } from 'icona';\n\n// Get suggestions as user types (like Lucide)\nconst suggestions = getIconSuggestions('che'); // Returns chevron icons\nconst moreResults = getIconSuggestions('arrow', 10); // Returns arrow icons\n\n// Advanced search with filters\nconst results = searchIcons({\n  query: 'message',\n  categories: ['Hicon'],\n  tags: ['communication'],\n  limit: 5\n});\n```\n\n## TypeScript Features\n\n### Icon Name Types\n\n```typescript\nimport type { IconName, IconComponentName } from 'icona/types';\n\n// Union type of all available icon names\ntype AllIcons = IconName; // 'home' | 'search' | 'chevron-right' | ...\n\n// Union type of all component names (PascalCase)\ntype AllComponents = IconComponentName; // 'Home' | 'Search' | 'ChevronRight' | ...\n```\n\n### Utility Types for Better Autocomplete\n\n```typescript\nimport type { IconsStartingWith, IconsContaining } from 'icona/types';\n\n// Get all icons starting with \"che\"\ntype ChevronIcons = IconsStartingWith<'che'>; // 'chevron-right' | 'chevron-left' | ...\n\n// Get all icons containing \"arrow\"\ntype ArrowIcons = IconsContaining<'arrow'>; // 'arrow-up' | 'arrow-down' | ...\n```\n\n## Search API\n\n### getIconSuggestions()\n\nPerfect for autocomplete dropdowns:\n\n```typescript\nfunction IconPicker({ query }: { query: string }) {\n  const suggestions = getIconSuggestions(query, 8);\n  \n  return (\n    <div>\n      {suggestions.map(icon => (\n        <div key={icon.name}>\n          <Icon name={icon.name} />\n          <span>{icon.description}</span>\n        </div>\n      ))}\n    </div>\n  );\n}\n```\n\n### searchIcons()\n\nAdvanced search with filters:\n\n```typescript\nconst results = searchIcons({\n  query: 'navigation',        // Search term\n  limit: 20,                  // Max results\n  categories: ['Linear'],     // Filter by category\n  tags: ['interface'],        // Filter by tags\n  minScore: 0.3              // Minimum relevance score\n});\n```\n\n### Other Utility Functions\n\n```typescript\n// Get all available categories\nconst categories = getCategories(); // ['Hicon', 'waveicons', ...]\n\n// Get icons in a specific category\nconst hiconIcons = getIconsByCategory('Hicon');\n\n// Get all available tags\nconst tags = getAllTags(); // ['navigation', 'communication', ...]\n\n// Get all icon names\nconst allNames = getAllIconNames();\n\n// Get metadata for a specific icon\nconst metadata = getIconMetadata('home');\n```\n\n## IDE Integration\n\n### VS Code\n\nWith TypeScript enabled, you'll get:\n\n1. **Import Autocomplete**: Type `import { }` and get icon suggestions\n2. **JSDoc Tooltips**: Hover over any icon to see description and examples\n3. **Error Detection**: TypeScript will warn about non-existent icon names\n4. **Go to Definition**: Navigate to icon definitions\n\n### WebStorm/IntelliJ\n\nSimilar features with rich IntelliSense support.\n\n## Component Usage Examples\n\n### React\n\n```tsx\nimport { ChevronRight, Home, Search, MessageSquare }  />\n    </View>\n  );\n}\n```\n\n## Build Configuration\n\nTo enable autocomplete features, make sure your build generates the type definitions:\n\n```typescript\nimport { generateIcons } from 'icona';\n\nawait generateIcons({\n  config: {\n    output: {\n      directory: './dist',\n      react: true,\n      vue: true,\n      reactNative: true\n    }\n  }\n});\n```\n\nThis generates:\n- Component files with JSDoc documentation\n- TypeScript definition files\n- Search index and metadata\n- Runtime search API\n\n## Comparison with Lucide\n\n| Feature | Lucide | Icona **Use TypeScript**: Enable strict mode for best autocomplete\n2. **Update IDE**: Use latest VS Code or WebStorm for optimal experience\n3. **Import Specifically**: Import only the icons you need for better tree-shaking\n4. **Use Search**: For large icon sets, use search functions instead of browsing\n5. **Check JSDoc**: Hover over icons to see descriptions and examples\n\n## Troubleshooting\n\n### Autocomplete Not Working\n\n1. Ensure TypeScript is enabled in your project\n2. Check that type definitions are generated in `/dist/types/`\n3. Restart your TypeScript language server\n4. Verify your `tsconfig.json` includes the types\n\n### Icons Not Found\n\n1. Run the build process to generate components\n2. Check that your icon names match exactly (case-sensitive)\n3. Verify the icon exists in your source directory\n\n### Search Results Empty\n\n1. Ensure icons have been processed with metadata\n2. Check that your query matches keywords or descriptions\n3. Lower the `minScore` threshold in search options
